Rasterization

During my 3D algorithmics classes, we had to build a software to understand better rasterization. This software had to have multiple features asked by our professor, and we were to do it on Android or Java. I choose Android and I'm going to show the features I have implemented through little cards below.

Base rendering and Commands

First of all, we were asked to do a Gouraud shader, with some basic object such as planes and cubes. In addition to that, your application shoul allow the user to move through the scene, to rotate the camera and I had a last control to reset the view and position of the user. So, to move, use the left part of the screen. To rotate the camera, use the right part of the screen. And to reset yourself, put three finger on the screen.

Parametric and ico spheres

Then we made more complex object, such as sphere. And we see two version of them, one made with a parametric equation. This technique is often reference as the parametric sphere and you found it in numerous software because it let you control much more easily the number of faces. The sceond one, less used, is the Ico-sphere. The goal is to start from an octahedron, and subdivide the face in four and put the new points to the same distance of the center. It less used, because even if it has better result from my point of view, you can't control the number of face. On the picture, the left object is an Ico-sphere with 4 division, and on the right is a parametric sphere with phi = 40 and theta = 40.

Import objects and reflection

This was one of the main features we had to make. The goal here was to import in our application an 3D object not generated inside our application. We had only one restriction, we needed to use the .obj format. Then we made reflection, to do so we simply put the ground a little bit translucid, and we render all the other object with the inverse modelMatrix.

Shaders Updates

After thoses points, we were asked to have better shaders. We had to make Lambert and Phong shaders and we had to make a system to switch shaders in the project (A container for shaders).

Textures

Then the last upgrade we had to make is adding texture to the phong shader, still with the switching system. That's the last update we were asked to do. If you want to test this project, here's a link to download the apk.
